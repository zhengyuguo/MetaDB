How to search for public experiments

    Search by keyword

    Enter a keyword in the query box at the top-left corner of home page. The keyword could be accession numbers, disease names, gene symbols, tissue types or any other terms of interest. Leave it blank if you do not want to search with a keyword.

    Search by gene name or disease name or tissue type

    To do a more specific search on gene symbols, disease names and tissue types, you can select the terms provided by the drop-down boxes and then submit the requests. Leave the corresponding drop-down box blank if you do not want to use the criteria to query the database.


    Relationships between different criteria

    The relationships between Gene, "Disease" and "Tissue" are OR. For example, if you choose Gene "Adar", Disease "Acute Lung Injury" and Tissue "293T cell", the database will return all the data set that are related with "Adar" OR "Acute Lung Injury" OR "293T cell".


    The relationship between "keyword" with the other three criteria ("Gene", "Disease" and "Tissue") is AND. For example, if you choose Gene "Adar", Disease "Acute Lung Injury" and Tissue "293T cell", and enter keyword "immune", then the database will filter the returned data sets by Gene "Adar", Disease "Acute Lung Injury" and "Tissue 293T cell" using keyword "immune".


    Filter your search

    After a successful search, you can even filter it with typing any word in the search box at the right side of home page to show the entries containing that word.

    To sort the table click the caret icon available at every column heading.

Why Use RNASeqMetaDB: Case studies

    The following two case studies using two common diseases demonstrate the advantage of using RNASeqMetaDB instead of ArrayExpress, a major public repository of RNA-seq data. The comparisons suggest that our manual curation does improve the annotation of gene targeted mouse RNA-seq data.

    Case 1: Asthma

    By searching “asthma” filtered by organism “Mus musculus”, experiment type "rna assay” and experiment type "sequencing assay" in ArrayExpress, one data set was returned as shown in the following screenshot. There are mouse ChIP-Seq data and human RNA-Seq data in this data set, but there is no mouse RNA-Seq data. It appears that ArrayExpress only looks for “mouse” and “RNA-Seq” but does not check whether there are indeed mouse RNA-seq data in the results.


    In comparison, as shown in the following screenshot, the query of “Asthma” in the “Disease” drop-down list in RNASeqMetaDB returned two data sets (E-GEOD-45975 and E-GEOD-49047). The first data set E-GEOD-45975 is related to asthma because it contains RNA-seq data from Bach2 knock-out and polymorphisms in this gene are associated with autoimmune and allergic diseases including asthma. The second data set E-GEOD-49047 is also related to asthma because the data is for studying the effect of DDAH1 overexpression of an asthma mouse model. Both of the data sets are manually annotated as related to asthma but missed by ArrayExpress.


    Case 2: Alzheimer’s disease

    By searching “Alzheimer’s disease” filtered by organism “Mus musculus”, experiment type "rna assay” and experiment type "sequencing assay" in ArrayExpress, four data sets are returned as shown the following screenshot.


    As shown in the following screenshot, the query of “Alzheimer disease” in the “Disease” drop-down list in RNASeqMetaDB returned three data sets.


    ArrayExpress identified an amyotrophic lateral sclerosis (ALS) related data set E-GEOD-43366 as Alzheimer’s disease related. The reason that ArrayExpress returns this data set is because the phrase “Alzheimer’s disease” appears in the data set description (the description says “These ALS microglia showing substantial induction of a ‘neurodegeneration-tailored phenotype’, with induction of lysosomal, RNA splicing, and Alzheimer's disease pathway genes”). RNASeqMetaDB does not return this data set.

    ArrayExpress identified E-GEOD-52024. This is a SuperSeries containing two SubSeries E-GEOD-52022 and E-GEOD-52023, among which only E-GEOD-52023 is an RNA-Seq data set. But ArrayExpress was not able to find this fine level of information. In contrast, RNASeqMetaDB accurately found the Alzheimer’s disease related data set E-GEOD-52023. This use case demonstrates that RNASeqMetaDB is able to provide data set information with fine granularity.

    Besides, E-GEOD-58261 was found in RNASeqMetaDB but missed in ArrayExpress. E-GEOD-58261 contains RNA-Seq data from the mouse model of Dnmt1 conditional knock-out and the publication related with this data set suggests that Dnmt1 regulates common genes related with Alzheimer’s disease.

    Both ArrayExpress and RNASeqMetaDB found E-GEOD-65159 as an Alzheimer’s disease related data set. In this data set, an Alzheimer’s disease mouse model was used to study the transcriptional dynamics.

    E-ERAD-160 is currently not included in RNASeqMetaDB because there is not a publication associated with it which prevents us from manually curating more detailed information. Once such information is available, this data set will be included in RNASeqMetaDB along with detailed curation information.
